# ClipForge 功能规划与后续方向

> 基于当前代码库（`src-tauri/src/*`、`src/App.tsx`）和已有记忆整理。

## 一、已具备的核心能力

- 剪贴板监听：文本 + 图片（500ms 轮询）
- 跨平台后端：Tauri 2 + Rust，`arboard` 负责剪贴板，`enigo` 模拟粘贴
- 系统托盘驻留、左键显示窗口、右键菜单
- 无边框窗口 + 自定义标题栏 + 失焦自动隐藏
- 全局快捷键（Ctrl+Shift+V）+ 快捷键冲突检测 + 录制配置
- 内容自动分类：URL、邮箱、代码、颜色、图片、文本
- 搜索 + 分页加载（20 条/页）
- 自定义分组（增删改、颜色标记）
- 分组筛选 + 类型筛选
- 设置页：开机自启、快捷键、数据库路径、历史记录上限、分组管理
- 历史记录限制：最多保留条数 / 最长保存天数（仅对未分组记录生效）
- 图片以 PNG 保存到 `db/images/`，缩略图 base64 返回

---

## 二、当前明显的短板

1. **性能隐患**
   - `list_items_by_type` 每次读取前 500 条在内存中过滤，数据量大会卡。
   - 内容类型在每次读取时动态计算，没有数据库索引。
   - 图片是逐一 `invoke("get_image_base64")` 加载，不是懒加载/流式。

2. **数据模型不完整**
   - 一条记录只能存「文本」或「图片」其一，不能同时存多种格式。
   - HTML 富文本字段已预留，但 `arboard` 跨平台不支持读取，目前写死为空。
   - 不支持文件（file copy）剪贴板。

3. **交互体验不足**
   - 列表没有键盘导航（上下键、Enter 复制、Delete 删除）。
   - 点击条目一定是「复制并粘贴」，无法只复制不粘贴。
   - 长文本无法展开预览；图片没有放大查看。
   - 没有「置顶 / 收藏」功能，重要记录容易被冲走。
   - 全局禁用了右键菜单，但条目内没有替代操作入口。

4. **数据管理薄弱**
   - 没有导入 / 导出 / 备份。
   - 没有清理统计（已删除多少、占用多少空间）。
   - 没有单条大小限制（大图可能撑爆数据库）。
   - 删除、清空不可逆，没有回收站/撤销。

5. **健壮性**
   - 多处 `lock().expect()` 和 `unwrap()`，Rust 端 panic 会直接带崩整个应用。
   - 数据库操作没有超时/重试；图片读取失败未处理。
   - 粘贴模拟在部分应用（UWP、管理员权限窗口）可能失效，没有 fallback。

## 三、建议优先级

### 高优先级（建议接下来就做）

| 功能 | 理由 | 大概改动点 |
|------|------|-----------|
| **键盘导航** | 快捷键呼出后能用 ↑/↓ + Enter 选条目，才是效率工具 | 前端 list 焦点管理、监听 keydown、调用 `copy_and_paste` |
| **复制但不粘贴** | 用户经常只想把内容放回剪贴板，不自动注入 | 右键菜单 or 按钮 + `copy_to_clipboard` 命令 |
| **置顶/收藏（Pin）** | 把常用片段固定在顶部，胜过翻历史 | 新增 `is_pinned` 字段 + 列表排序 + 设置入口 |
| **优化按类型筛选** | 500 条内存过滤是性能炸弹 | 在 `clipboard_items` 表中增加 `content_type` 列并持久化，加索引 |
| **图片懒加载/缩略图** | 图片多时会阻塞主界面 | 前端 Intersection Observer + 后端返回压缩缩略图 |

### 中优先级（完善产品体验）

| 功能 | 理由 | 大概改动点 |
|------|------|-----------|
| **导入/导出（JSON/SQLite）** | 换电脑、备份必备 | 后端读文件复制/写 JSON，前端选择路径 |
| **HTML/富文本支持** | 从浏览器复制带格式文本时保留样式 | 等 `arboard` 支持 or 平台原生 API 临时方案 |
| **文件剪贴板支持** | 复制文件后在 ClipForge 中保留路径 | 监听文件列表格式（Windows CF_HDROP / macOS NSFilenames） |
| **内容预览弹窗** | 长代码、长链接、图片需要查看全貌 | 前端 modal + 后端提供完整内容 |
| **回收站/撤销删除** | 误删清空的救命功能 | 新增 `deleted_at` 软删除表，30 天后自动清理 |
| **历史记录统计** | 让用户知道占用了多少空间 | 后端计算条目数、图片总大小、最早记录 |

### 低优先级（锦上添花）

| 功能 | 理由 | 大概改动点 |
|------|------|-----------|
| **端对端加密/数据库加密** | 敏感数据保护 | 引入 `sqlcipher` 或应用层加密 |
| **自动更新** | 发布后才需要 | Tauri 的 updater 插件 |
| **使用统计/错误上报** | 了解用户行为 | 可选、本地匿名 |
| **跨平台光标定位补齐** | macOS/Linux 当前只能居中 | 用平台原生 API 获取光标/焦点信息 |
| **插件/脚本扩展** | 高级用户自定义转换 | 设计脚本 API，较复杂 |

## 四、推荐的最小下一版（MVP 增量）

如果只想先做一个版本，建议按这个顺序：

1. 给 `clipboard_items` 增加持久化的 `content_type` 列和索引，淘汰 500 条内存过滤。
2. 列表键盘导航：呼出窗口后 `↓` 进入列表，`↑/↓` 选择，`Enter` 复制粘贴，`Esc` 隐藏，`Delete` 删除。
3. 增加「仅复制」按钮 / 右键菜单项，不自动粘贴。
4. 增加 `is_pinned` 字段，支持把条目置顶，置顶项不受历史限制清理。
5. 图片懒加载：滚动到可视区域再请求 `get_image_base64`，减少初始加载。

这 5 项做完，ClipForge 的“顺手程度”会有质变，且不会引入大的架构改动。
